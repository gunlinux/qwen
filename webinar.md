# документация

У хорошего промта просто в чате есть три пункта:
context, (работа с контекстом - это кодовая база, AGENTS.md и документация проекта) 
входные данные (данные которые он смотрим) + ПРОМПТ
выходные данные (что получилось)

Самое сложное при работе с проектом - это познакомить агента с кодовой базой и  актуализировать знания модели, текущего состояния памяти:

Пример: условный uv или определенная версия фреймворка - у всех llm много знаний о sqlalchemy 1 версии, а про второй гораздо меньше кода.Поэтому можно форсировать использовать `uv вместо pip`, `pyproject.toml вместо requiments.txt` или context7 помогает

/init

## Работа с памятью:

/memory add --global - можно докинуть общие знания для разработки на конкретном языке, codestyle команды или фреймворка /memory add --project - дока конкретного проета - описания teststyles, моделей, роутов, внешних систем
/memory refresh перечитываем знания
/clear
/compress - для долгих чатов
